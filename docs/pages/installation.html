Installation Guide

Get VectorReVamp up and running in your development environment

System Requirements

Python

  • Python 3.8+
  • Recommended: Python 3.10+

Memory

  • Minimum: 4GB RAM
  • Recommended: 8GB+ RAM
  • For parallel processing: 16GB+ RAM

Storage

  • Minimum: 1GB free space
  • Recommended: 5GB+ for templates and data

Operating System

  • Linux (Ubuntu 18.04+)
  • macOS (10.15+)
  • Windows (10+ with WSL)

Installation

Option 1: Install from PyPI

# Install VectorReVamp
pip install vectorrevamp

# Or install with all optional dependencies
pip install vectorrevamp[all]

Option 2: Install from Source

# Clone the repository
git clone https://github.com/SWORDIntel/DSMILSystem.git
cd DSMILSystem/tools/VectorReVamp

# Install in development mode
pip install -e .

# Or install with all optional dependencies
pip install -e .[all]

Optional Dependencies

# For full functionality
pip install vectorrevamp[all]

# Individual extras:
# pip install vectorrevamp[analysis]     # Advanced static analysis
# pip install vectorrevamp[parallel]     # Parallel processing
# pip install vectorrevamp[plugins]      # Plugin system
# pip install vectorrevamp[testing]      # Additional testing tools
# pip install vectorrevamp[docs]         # Documentation generation

Quick Start

1. Generate Tests for a Python Module

# Generate tests for a Python file
vectorrevamp generate my_module.py

# Generate tests for an entire package
vectorrevamp generate my_package/ --recursive

# Specify output directory
vectorrevamp generate my_module.py --output tests/

2. Generate Tests with Advanced Options

# Use templates and quality validation
vectorrevamp generate my_module.py --use-templates --quality-threshold 0.8

# Enable parallel processing
vectorrevamp generate my_package/ --parallel --max-workers 4

# Load custom plugins
vectorrevamp generate my_module.py --plugin-path ./my_plugins/

3. View Generated Tests

# Run generated tests
pytest tests/

# View coverage report
pytest tests/ --cov=my_module --cov-report=html
open htmlcov/index.html

Detailed Setup

Configuration File

Create a vectorrevamp.json configuration file:

{
  "language": "python",
  "framework": "pytest",
  "use_templates": true,
  "template_quality_threshold": 0.7,
  "parallel_enabled": true,
  "max_workers": 4,
  "output_directory": "tests/",
  "plugin_paths": ["./plugins/"],
  "coverage_targets": ["functions", "branches", "lines"],
  "quality_checks": ["structural", "functional", "syntactic", "coverage"]
}

Environment Variables

# Set environment variables
export VECTORREVAMP_CONFIG=path/to/config.json
export VECTORREVAMP_LOG_LEVEL=INFO
export VECTORREVAMP_CACHE_DIR=~/.cache/vectorrevamp

Plugin Development Setup

# Create plugin directory structure
mkdir -p my_plugins/language_plugins/
mkdir -p my_plugins/framework_plugins/
mkdir -p my_plugins/domain_plugins/

# Create a custom language plugin
cat > my_plugins/language_plugins/custom_lang.py << 'EOF'
from vectorrevamp.plugin_system import LanguagePlugin

class CustomLanguagePlugin(LanguagePlugin):
    name = "custom_lang"
    extensions = [".cl", ".custom"]

    def analyze_file(self, file_path):
        # Custom analysis logic
        pass

    def generate_test_template(self, function_info):
        # Custom template generation
        pass
EOF

Verification

Check Installation

# Verify installation
vectorrevamp --version

# Check available commands
vectorrevamp --help

# Test basic functionality
vectorrevamp generate --help

Test Generation

# Create a simple test file
cat > test_example.py << 'EOF'
def add_numbers(a, b):
    """Add two numbers together."""
    return a + b

def multiply_numbers(a, b):
    """Multiply two numbers."""
    return a * b
EOF

# Generate tests
vectorrevamp generate test_example.py --output generated_tests/

# Run generated tests
pytest generated_tests/ -v

Troubleshooting

Import Error

Issue: ModuleNotFoundError: No module named 'vectorrevamp'

Solution: Ensure VectorReVamp is installed and Python path is correct.

pip install vectorrevamp
python -c "import vectorrevamp; print('Import successful')"

Permission Error

Issue: Cannot write to output directory

Solution: Check permissions and create directory if needed.

mkdir -p output_directory/
vectorrevamp generate input.py --output output_directory/

Memory Error

Issue: Out of memory during large codebase analysis

Solution: Reduce parallel workers or process in batches.

vectorrevamp generate large_codebase/ --max-workers 1
# Or process specific files
vectorrevamp generate specific_file.py

Plugin Loading Error

Issue: Custom plugins not loading

Solution: Check plugin path and ensure plugins inherit from base classes.

vectorrevamp generate --plugin-path ./plugins/ --debug
# Check plugin inheritance
python -c "from my_plugin import MyPlugin; print(MyPlugin.__bases__)"
